# AGENTS

## Dev environment tips
- The main run environment is PowerShell 7.

## Dev tips
- Update the script document after any code is updated.
- All functions written by AI models must include a comment in the function's documentation comment indicating the model ID, in the format: `# Generated by AI: <ModelID>`. Example: `# Generated by AI: GPT-5.3`.

## Files in `/bin` directory

`/bin` directory contains the entry-point PowerShell scripts for this repository and is added to the user's `PATH`. The following rules apply to new or modified files in `/bin`:

- All `.ps1` files under `bin` must configure `Set-StrictMode -Version 3.0`.

## Files in `/lib` directory

`/lib` directory contains PowerShell modules that are imported by scripts in `/bin`. The following rules apply to new or modified files in `/lib`:

- All `.psm1` files under `lib` must configure `Set-StrictMode -Version 3.0`.
- Functions in `.psm1` files under `lib` must include `OutputType` attribute by default, unless the user explicitly says it is unnecessary.

## Testing
- Unit tests use the Pester framework.
- Test files are placed next to the file they test, with a `.Tests.ps1` suffix (e.g., `opencode-models.psm1` → `opencode-models.Tests.ps1`).
